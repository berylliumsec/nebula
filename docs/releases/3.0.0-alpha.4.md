# Nebula 3.0.0-alpha.4

Nebula 3.0.0-alpha.4 supersedes the unpublished alpha.2 and alpha.3
candidates. It contains their Apple-silicon release corrections plus the final
installer-inspection and packaged-sidecar cleanup fixes found during protected
release validation.

## Changes since alpha.1

- The protected macOS release path targets Apple silicon only and pins Apple's
  system `/usr/bin/xattr` during Tauri packaging.
- Licensed Tauri DMGs are accepted non-interactively during preparation and
  finalization, while their signatures, contents, and mounted applications
  remain fully inspected.
- The packaged desktop self-test gives Nebula Core a bounded graceful-shutdown
  window after closing its lifetime pipe before falling back to process-group
  signals. Cleanup is idempotent if Core exits during that boundary.
- Tall terminal screenshots can fit completely inside the image editor at
  zoom levels below ten percent without enlarging smaller images.
- Diagnostics and container-terminal tests synchronize on observable state,
  and UI tests share the release workflow's bounded timeout.
- The standalone release-credential smoke workflow has been removed; the
  protected release itself remains the authoritative credential validation.
- Finalization, updater publication, checksums, SBOM counts, and release copy
  consistently describe the Apple-silicon-only macOS artifact set.

There are no intentional engagement-data format changes from alpha.1. No
alpha.2 or alpha.3 release was published.

## Native packages

The release workflow stages the following artifacts for manual review:

- signed and notarized macOS 13+ DMGs for Apple silicon, in direct and managed
  variants;
- a Linux x86_64 AppImage for direct distribution; and
- a Linux x86_64 DEB for managed distribution.

Direct builds can consume Nebula's signed updater channel. Managed builds omit
the direct updater so deployment remains under administrator control. Windows,
Intel macOS, and Linux arm64 packages are not included in this alpha.

Docker or Podman is required for terminal and automation features. A model
provider is optional; the human terminal, evidence workflow, notes, findings,
and reports remain usable without one.

## Alpha limitations

- This is a prerelease and may contain compatibility defects or incomplete UX.
- Engagement bundle export is integrity-manifested, but this alpha does not yet
  provide a bundle restore path.
- Terminal and automation availability depends on a compatible local container
  runtime and its networking configuration.
- Local and hosted model capabilities vary. Nebula enables structured tools
  only after the selected provider/model passes its verification contract.
- Browser persistence differs on macOS 13 because that WebKit version does not
  support the named persistent stores used on newer systems.

## After installation

Launch the desktop with `nebula`. Confirm the bundled Core and local runtime
boundary before beginning an engagement:

```console
nebula-core doctor --json
```

Report defects with the exact version, platform, failed action, correlation ID,
and a Diagnostics support bundle after reviewing it for sensitive engagement
data.
